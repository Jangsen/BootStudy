5장 마무리 요약

1. 데이터 조회 과정

웹 페이지에 게시글을 등록하면 서버를 통해 DB에 저장됨

DB에 저장된 데이터는 다음 과 같은 과정을 거쳐 웹 페이지에서 조회할 수 있음

1) 사용자가 웹 페이지에서 데이터를 조회해 달라고 URL 요청을 보냄

2) 서버의 컨트롤러가 이 요청을 받아 해당 URL 에서 찾으려는 데이터 정보(여기서는 id)를
레포지토리에 전달함

3) 레포지토리는 정보(id)를 가지고 DB에 데이터 조회를 요청함

4) DB는 해당 데이터를 찾아 이를 엔티티로 반환

5) 반환된 엔티티는 모델을 통해 뷰 템플릿으로 전달

6) 최종적으로 결과 뷰 페이지가 완성돼 사용자 화면에 출력


2. @PathVariable

URL 요청으로 들어온 전달값을 컨트롤러의 매개변수로 가져오는 어노테이션

3. findById()

JPA의 CrudRepository가 제공하는 메소드로, 특정 엔티티의 id 값을 기준으로 데이터를 찾아
Optional 타입으로 반환

4. findAll()

JPA의 CrudRepository가 제공하는 메소드로, 특정 엔티티를 모두 가져와 Iterable 타입으로 반환함

5. {{#artilcs}}{{/artilce}}

뷰 페이지에서 모델에 등록된 article의 사용 범위를 지정할 때 사용하는 머스테치 문법

{{#article}} 부터 {{/article}}까지 범위 내에서 article 데이터를 사용할 수 있음

등록된 데이터가 복수이면 해당 범위 코드가 반복됨

6. 반환 데이터 타입 불일치 문제 해결방법

특정 메소드가 반환하는 데이터 타입과 사용자가 작성한 반환 데이터 타입이 다를 경우, 3가지 밥업으로 해결 가능

1) 메소드가 반환하는 데이터 타입을 사용자가 작성한 데이터 타입으로 캐스팅(형변환) 하기

2) 사용자가 작성한 데이터 타입을 메소드가 반환하는 데이터 타입으로 수정하기

3) 메소드의 반환 데이터 타입을 원하는 타입으로 오버라이딩하기
